# みなさん情熱を持って「異議あり！」と叫んでいますか？

# ⚖️ メン・トライアル：就活逆転裁判 (Men-Trial)

- 面接中の就活生と面接官の「生体・音声・発言内容」をリアルタイムで解析し、画面上に**「逆転裁判」風のオーバーレイUIとして可視化するデスクトップアプリ**
- 一方的な「尋問」になりがちな面接を、ゲーム感覚で戦い抜く「対戦型法廷バトル」へ変貌させ、就活生の緊張をアドレナリンに変えるプロダクト。

---

## 💡 機能内容

### ⚡ リアルタイム雷バナー（コア機能）
- 面接官の画面（デスクトップキャプチャ）と就活生のカメラ・マイクからリアルタイムに優勢度を算出。
- 画面上部で就活生（左・青）と面接官（右・赤）の雷がせめぎ合い、戦況に応じて衝突点がダイナミックに左右にスライド。
- 視覚的に「今、自分がどれくらい主導権を握れているか」が1Hzのリアルタイムでフィードバックされます。

### 💥 「異議あり！」必殺技カットイン
- 就活生が論理的で素晴らしい返答をしたり、面接官の笑顔を引き出して優勢度が閾値（デフォルト $80\%$）を超えた瞬間、画面にド派手な**「異議あり！」風カットインアニメーション**（Rive）が発生。
- 同時に重厚なSE（howler.js）が鳴り響き、極限状態の就活生の脳汁と自信を最大限に分泌させます。

### 👁️ カンペ偽装アイコタクトシステム
- 就活生が画面上の別ディスプレイのカンペを読んでいる（視線がカメラから外れている）のを検知。
- **虹彩トラッキング（MediaPipe Iris）**により、瞳の位置をリアルタイムで画像変形（WebGL）し、面接官のカメラには**「常にカメラ正面を力強く凝視している」**ようにディープフェイク補正します。
- カンペを読むための視線のゆらぎを、高度な画像処理技術で完全に隠蔽する「最強のカンニングアシスト」です。

### 📈 非接触心拍推定（rPPG）による「真の焦り」可視化
- Webカメラ映像から、額や頬の皮膚の微細な色変化（血流の脈動）を画像解析で抽出。
- 外部デバイスを一切身につけることなく、**非接触でリアルタイム心拍数（BPM）を測定**。
- 「早口になった」「言葉が詰まった」といった表層的な焦りだけでなく、面接官の鋭い質問による「自律神経の乱れ（心拍数の跳ね上がり）」を科学的な「焦りスコア」として優勢度に反映。

### 🎙️ リアルタイム音声ディフェンス & 判定プロンプター
- 就活生の声が緊張で上ずった場合、`AudioWorklet` を用いて自動でピッチや周波数特性を補正し、「堂々としたプロっぽい低音」にリアルタイムでボイス加工してZoom等に送出。
- さらに、面接官の発言をリアルタイムでテキスト化（STT）し、裏でGemini 1.5 Flashが「就活生のキャラクター設定」に基づいた最適な回答スクリプトを生成、プロンプターとして画面にスクロール表示します。

## 🚀 推しアイデア

### 表情・声・心拍・ロジックを融合する「マルチモーダル優勢度（Dominance Score）」
従来の面接練習ツールや感情分析ソフトは「笑顔の回数」や「発話速度」などを個別にグラフ表示するだけでした。  
「メン・トライアル」は、それらの多層的な生体・論理データを単一の「戦況（優勢度）」にマージして演出に還元します。

```
【優勢度 (Dominance Score) の重み付け合成】
🟦 就活生の笑顔・表情の余裕 (30%)
🟥 面接官の表情変化・笑顔引き出し (15%)
🔊 声の焦り・ピッチ変動の少なさ (20%)
🚫 フィラー（「えっと」等）の少なさ (15%)
🧠 返答内容の論理的整合性（Gemini評価） (20%)
```

#### 「平常心で話した内容だけでは、面接官の心は動かせない。」
就活生が「論理的回答（Gemini: 90点）」を叩き出しても、声が震え心拍数が $140\text{ BPM}$ に跳ね上がっていれば、雷ゲージは押し戻されます。  
論理的な冷静さと、心身の堂々としたパフォーマンス。その両方が噛み合った瞬間だけ、雷バナーが右に振り切れ、脳汁のカットインが発動します。

## 🛠️ 使用技術

### フロントエンド（オーバーレイ / リアルタイム解析）
- **Vite 6** / **React 19** / **TypeScript 5**
- **Zustand** (超軽量・低遅延の優勢度状態管理)
- **MediaPipe Face Landmarker & Iris** (顔の焦り・表情・視線虹彩トラッキング)
- **Meyda** (マイク音声からのピッチ変動・発話速度・ポーズ比抽出)
- **WebGL / Canvas API** (rPPG心拍推定のための皮膚ピクセル解析、瞳の強制アライメント補正)
- **Rive / framer-motion** (ベクターアニメーション / 逆転カットイン演出)
- **howler.js** (3Dポジショナル対応 SE/BGM 再生)
- **Tailwind CSS v4** (デザインシステム・UIスタイル)

### バックエンド（Electronメインプロセス & AI）
- **Electron 34** (背景透過・常に最前面・ホバー時のみクリック透過解除のオーバーレイシェル)
- **electron-audio-loopback** (macOSの ScreenCaptureKit を用いた、面接官の出力音声の直接キャプチャ)
- **Gemini 1.5 Flash API** (応答の適切さ評価、`responseSchema` による構造化JSON出力の強制)
- **Deepgram Streaming API / Gemini Live STT** (超低遅延リアルタイム音声テキスト化)
- **Web Audio API (AudioWorklet)** (就活生のマイク音声に対するリアルタイムピッチ補正)

## 📊 技術構成図

```
[就活生 (カメラ/マイク)] ────> [MediaPipe / Meyda (1Hz解析)] ─┐
                                                            │
[面接官 (画面/音声)] ───(SCKit)───> [MediaPipe (表情解析)] ───┼─> [Zustand Store] ─> [雷バナー/Rive UI]
                                                            │
[音声テキスト化] ────> [Gemini 1.5 Flash (論理判定)] ──────┘
```
- カメラとマイクの2系統生データ ↔ MediaPipe/Meydaによるフロントエンドでのリアルタイム特徴量抽出。
- 音声は `electron-audio-loopback` を通じてメインプロセスでキャプチャし、STT $\rightarrow$ Geminiへとパイプライン接続。
- メインプロセスからレンダラープロセスへIPC経由で解析結果を高速通知し、ZustandのStoreで優勢度を一元管理。

## 💪 頑張ったこと
- **OSネイティブ機能とWeb技術の融合**: Electronのウィンドウ設定をミリ秒単位で制御し、コントロール部分（ボタンや入力フォーム）にマウスホバーした時だけクリック透過を無効化（インタラクティブにする）する「ストレスフリーな透過UI」を実現。
- **Webカメラだけでの非接触生体計測（rPPG）**: 額の肌の色（赤血球の吸光度変化）から心拍数をフィルタリングするためのIIRフィルタ・FFTロジックを、フロントエンドの軽量なJS/TSのみで破綻なく実装。
- **4言語・多重パイプラインの結合**: 音声ループバック(C++/Swift/RustのScreenCaptureKitラッパー)、Electronメイン(Node.js/TS)、フロント解析(MediaPipe/JS/Canvas)、AI(Python/Gemini API)の異なる技術層を、型安全なIPC（Inter-Process Communication）で1本の低遅延リアルタイム・ストリーミングに繋ぎ合わせたこと。

## 🎜 難しかったこと
- **描画とリアルタイム解析の並列処理（FPS低下問題）**:
  - MediaPipeとMeydaを同時に動かしながら、WebGLのテクスチャ補正とRiveのアニメーションを描画すると、ブラウザプロセスのCPU/GPUがボトルネックになり画面がカクつく問題。
  - **解決策**: 重い顔ランドマークトラッキングとrPPGフィルタの処理を **Web Workers** に逃がし、メインスレッド（描画側）のFPSを $60\text{fps}$ に維持。
- **フィラー検出のリアルタイム性と日本語フィラーの「整形問題」**:
  - LLMや高度なSTTは、気を利かせて「えっと」「あのー」などのフィラーを自動で削除してテキスト化してしまう問題。
  - **解決策**: STTプロバイダの設定を調整し、フィラーをそのまま残して出力する streaming モードを特定。発話中のバッファに対して正規表現および形態素解析をかけ、「えっとですね」の瞬間的な検知率を向上。
