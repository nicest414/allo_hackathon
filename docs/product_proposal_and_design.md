# 就活面接 優劣判定オーバーレイ（仮）総合企画・設計書

本ドキュメントは、「就活面接 優劣判定オーバーレイ」プロジェクトのプロダクト概要、技術構成、ユーザー体験フロー、ゲーミフィケーション要素、およびハッカソンでの技術評価を最大化するための「技術の無駄遣い」の提案について総合的にまとめたものです。

---

## 🧭 1. プロダクト概要
面接中の緊張感をゲーム化（エンタメ化）し、就活生が主体的に楽しみながら面接に挑めるようにするための**「逆転裁判」風のデスクトップ・オーバーレイアプリ**です。

### 💡 主要機能
- **常に最前面の透過オーバーレイ**: 会議ツール（Zoom, Meet等）に重ねて表示され、操作を阻害しない（クリック透過制御）。
- **リアルタイム優勢度判定**: 就活生と面接官の「表情・声・発言内容」をリアルタイム解析し、0〜100の数値として算出。
- **せめぎ合う雷UIバナー**: 画面上部で両者のパワーバランスが雷の衝突として視覚化され、優勢な側がエリアを押し広げる。
- **「異議あり！」風カットイン演出**: 優勢度が一定ライン（例：80%）を超えた瞬間に、ド派手なアニメーションと効果音でプレイヤー（就活生）の脳汁を分泌させます。

---

## 🛠️ 2. 現在のシステムアーキテクチャ & 技術スタック
Electronのメインプロセス（Main）とレンダラープロセス（Renderer）で安全に責務を分離しています。

```
src/
├── main/               # OS API (ScreenCaptureKit等) や APIキー等の秘匿情報の管理
├── preload/            # レンダラーに安全なIPCブリッジを公開
├── renderer/           # React UI / メディア取得 / 特徴量抽出 / スコア計算
└── shared/types/       # メイン・レンダラー間のIPC型定義
```

### 💻 技術選定一覧
- **デスクトップシェル**: `Electron`（背景透過・常に最前面・クリック透過ウィンドウの制御）
- **音声ループバック**: `electron-audio-loopback` (macOS専用のScreenCaptureKitを使用し、面接官の声をキャプチャ)
- **UI / フロントエンド**: `Vite` + `React` + `TypeScript` + `Zustand` (状態管理)
- **演出ライブラリ**: `Rive` (ベクターアニメーション), `framer-motion` (UIトランジション), `howler` (音響)
- **顔分析**: `MediaPipe Face Landmarker`（就活生と面接官の表情・笑顔・焦りトラッキング）
- **声分析**: `Meyda`（就活生の声のピッチ変動・発話速度・ポーズ比のリアルタイム抽出）
- **判定AI**: `Gemini 1.5 Flash` (構造化出力 `responseSchema` を用いた返答内容の適切さ判定)

---

## 🏃 3. ユーザーフロー (利用の流れ)

1. **事前セットアップ**: 自分のカメラとマイク、およびキャプチャ対象の面接官の画面・音声ループバックを指定。
2. **面接スタート**: 会議アプリの上にオーバーレイが起動し、お互いの状態のトラッキングを開始。
3. **リアルタイムな攻防**: 表情が引きつったり、フィラー（「えっと」等）を言うと劣勢へ、笑顔を見せたり論理的な回答をすると優勢へスライド。
4. **必殺技（カットイン）**: 優勢度が限界を超えると、画面いっぱいにカットインが走りカタルシスを提供。
5. **面接終了**: 最大優勢度や、Geminiによる発言フィードバックを確認。

---

## 🎮 4. ユーザーが「面白い！」と感じるポイント（ゲーミフィケーション）

- **面接の「格闘ゲーム」化**: 防戦一方だった面接が「面接官のHPを削り、自分のゲージを維持する対戦アクション」になり、恐怖感がワクワク感に変わります。
- **「脳汁」カットイン**: 会心の回答時にRiveアニメーションと大音量の効果音（SE）が走ることで、ゲームで必殺技を決めた時のような強烈な達成感（カタルシス）を得られます。
- **リアルタイムセルフコントロール**: 「早口になっている」「フィラーが多い」などの指標が直感的に視覚フィードバックされるため、ゲーム感覚で話し方を修正できます。
- **面接官を「ボスキャラ」として攻略する視点**: 面接官の笑顔を引き出してゲージを押し返すため、面接官をコミュニケーションの「攻略対象」として捉えられます。

## 🚀 5. 技術の無駄遣い（Overengineering）＆ 技術レベル向上案
ハッカソンでの「技術評価点」を最大化するための、高度でユニークな拡張アイデアです。

### ⚡ 案1. Webカメラによる非接触心拍推定（rPPG）
- **概要**: 顔の微小な色変化（血流）から、心拍数（BPM）やストレス度をリアルタイム推定する。
- **技術**: MediaPipeの皮膚領域から緑（G）チャンネル平均値を抽出し、バンドパスフィルタ処理をしてBPM算出。
- **無駄遣い感**: 面接官の質問による「ごまかせない生体的な焦り（自律神経の乱れ）」を科学的に可視化。

### 🛰️ 案2. WebRTCによる「リアルタイム・セコンド（カンペ）システム」
- **概要**: 面接の映像・音声を友人のPCへ超低遅延ライブ配信し、友人が就活生の画面にリアルタイムでカンペを書き込む。
- **技術**: WebRTC datachannel（低遅延P2P配信）と WebSockets（双方向テキスト同期）。
- **無駄遣い感**: 面接を完全に「タッグマッチ」として捉え、裏で友人がセコンドとして指示を送るズルいシステム。

### 🧠 案3. ONNX Runtime Web による「ローカル音声感情認識 (SER)」
- **概要**: ユーザーのローカルPCのGPUをぶん回し、ブラウザ上で直接AI（音声感情認識モデル）の推論を走らせる。
- **技術**: `onnxruntime-web` (WebGPU/WASM) を用い、`Wav2Vec2` などの感情分類モデルをローカル実行。
- **無駄遣い感**: クラウドAPIを使わず、PCのファンを爆音で回しながらローカルAI推論をするという狂気。

### 👾 案4. Three.jsによる「3D格闘ゲーム風アバターバトル」
- **概要**: 画面の隅で、就活生と面接官の表情に完全同期した3Dモデルが格闘ゲームのように殴り合う。
- **技術**: Three.js / React Three Fiber を用い、MediaPipe Face Mesh から得た頭部回転・表情を3Dアバターのボーンに同期（表情キャプチャ）。
- **無駄遣い感**: 真面目な面接の裏で、3Dアバターが必殺技を放ち合っているというシュールさと高度なCG技術の融合。

### 🎙️ 案5. AudioWorkletによる「リアルタイム音声ディフェンス（ボイス加工）」
- **概要**: 就活生の声が焦りや緊張で上ずった際、自動的に周波数成分を補正して「堂々としたプロっぽい声」にリアルタイムで加工してZoom等に送出する。
- **技術**: Web Audio API の `AudioWorklet` を用いた低遅延デジタル信号処理（DSP）、または WebAssembly によるリアルタイム声質変換。
- **無駄遣い感**: 面接の緊張が相手に伝わらないよう、声そのものをリアルタイムでディープフェイクするズル賢いシステム。

### 👁️ 案6. 虹彩トラッキングによる「カンペ偽装アイコタクトシステム」
- **概要**: 就活生が別画面のカンペを読んでいる（視線がカメラから逸れている）時でも、面接官側の画面には「常にカメラを凝視している」ように見せる。
- **技術**: MediaPipe Iris（虹彩検出）による精密な視線計測と、WebGLを用いたリアルタイムテクスチャ変形（瞳の位置をカメラ中心に強制補正）。
- **無駄遣い感**: カンペを読むための視線の動きを、高度な画像処理技術で完全に隠蔽する「無駄に完璧なカンニング行為」。

### 🔌 案7. Web Serial APIによる「物理的冷や汗フィードバックデバイス」
- **概要**: 面接の優勢度が低下（劣勢）するにつれて、就活生が座っている椅子に仕込んだバイブレーションが振動したり、首元のファンから冷風が噴射されるなど、ゲームの「被ダメージ」を現実世界にフィードバックする。
- **技術**: **Web Serial API** を使用し、ブラウザ（Electron）からUSB接続したマイコン（Arduino / ESP32）へ直接パルス信号を送り、物理ファンやモーターを動的に制御。
- **無駄遣い感**: 面接の精神的プレッシャーを、わざわざハードウェア回路を組んで「物理的なダメージ」に翻訳する過剰設計。

### 🔥 案8. マルチスレッドによる「最強返答プロンプター（ゴーストライター）」
- **概要**: 面接官が喋った質問のSTTテキストが流れた瞬間、裏でローカルの複数LLM（Llama, Gemma等）に同時に回答を生成させ、最も適合度の高い回答を自動ブレンドしてプロンプターにカンペとして流し出す。
- **技術**: Web Workers によるマルチスレッド制御、`transformers.js` を用いた複数軽量LLMの並列同時推論。
- **無駄遣い感**: 面接の数秒という猶予の間に、PCのGPUリソースを食い潰してAI同士に「最強の回答」を競わせる究極のカンニングプロンプター。
