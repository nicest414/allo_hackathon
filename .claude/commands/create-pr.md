以下の手順でGitHub Pull Requestを作成してください。

## 手順

### 1. 現在の状態を確認

以下を並列で実行してください：
- `git status` — 変更ファイルの確認
- `git log main..HEAD --oneline` — ブランチのコミット一覧
- `git diff main...HEAD` — mainブランチからの差分全体

### 2. PRタイトルと説明文を生成

差分を分析し、以下の形式でPRの内容を生成してください。

**タイトル**（70文字以内、変更の本質を端的に表す）

**説明文**（以下のセクションを含む）：
```
## 概要
（変更の目的と背景を2〜3文で説明）

## 変更内容
（変更点を箇条書きで列挙）

## テスト確認項目
（レビュアーが確認すべき点をチェックリスト形式で記載）

## 関連issue
close #（issue番号）
（関連issueが複数ある場合は `close #番号` を行ごとに追加）

🤖 Generated with [Claude Code](https://claude.com/claude-code)
```

ブランチ名が `issue/{番号}-...` の形式の場合、そのissue番号を自動的に `close #番号` に使用してください。

### 3. ユーザーに確認

生成したタイトルと説明文をユーザーに提示し、以下を確認してください：
- このタイトルと説明でPRを作成してよいか
- マージ先ブランチ（デフォルト: `main`）
- 変更が必要な場合はフィードバックを受け取る

**ユーザーが承認するまでPRを作成しないでください。**

### 4. PRを作成

ユーザーの確認が取れたら、以下を実行してください：

1. リモートにプッシュ済みか確認。未プッシュなら `git push -u origin HEAD`
2. `gh pr create` でPRを作成

```bash
gh pr create --title "タイトル" --body "$(cat <<'EOF'
説明文
EOF
)"
```

3. 作成されたPRのURLをユーザーに伝える
